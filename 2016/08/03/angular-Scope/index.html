angular Scope

发表于 2016-08-03   |  
Scope是什么
Scope 是一个对象，这个对象引用程序数据。他是表达式的执行上下文。Scope的层级结构和DOM结构很类似。Scope 可以监视 表达式和传播事件。


Scope的特点
scopees 提供API（$watch）观察模型的变化。
scopes 提供API($apply)可以把模型的改变传播进视图从angular范围()的外面。
Scopes可以内嵌来限制对属性的访问，也可以提供对共享模型属性的访问。
内嵌的有 子socpe，也有 隔离scope。子scope可以从父scope继承 属性。隔离的scope不行。

Scope作为数据模型
Scope是控制器和视图的胶水。在模板link阶段，指令在scope上面设置了 $watch 表达式。这个$watch允许指令被通知属性的变化，这有允许指令渲染更新的值到dom里面。

控制器和指令都对scope有引用，但不是彼此。这个安排吧控制器从指令和dom中隔离开了。


Scope 层级结构

每一个angular应用程序，都只有一个Root scope，但可以有许多子 scope。
应用程序可以有很多scope。有些directive可以创建子scope。当新的scope创建的时候，他们被添加做父scope的孩子。这样一来，一个和dom结构平行类似的树形结构就创建出来了。

当angular 执行的时候，他首先查找和指定元素有关联的scope。如果没有发现，查找父亲。。。直到找到 根scope。js中这种行为叫原型继承。

angular自动放置了一个 ng-scope类在和scope有关联的元素上面。

从dom元素获取scope
Scopes以 $scpoe 的数据属性附加到DOM，可以以调试的目的获取。ng-app指定了根scope的附加元素。

通过这个方法可以查看dom上附加的scope
angular.element($0).scope()

Scope的事件传播
Scopes可以以类似DOM事件的方式传播事件。 事件可以 broadcated到scope孩子也可以emitted父亲。

Scope 生命循环
自然流程里面，浏览器接受事件然后执行响应的回掉函数。一旦回调函数调用完成，浏览器会重新渲染dom，并且返回等待更多的事件。

当浏览器调用angular执行上下文外面的js的时候，angular不知道model的修改。为了合适的模型修改，执行上下文必须进入angular的执行上下文，用$apply.只有在apply方法内部的模型修改，angular才会考虑在内。

在评估表达式之后，apply方法处理了一个$digest.在$digest 阶段，scope测试所有的$watch表达式并且和前面的值对比。这个脏检查以异步的方式进行。这意味着一个赋值语句$scope.username="angular" 不会立即引起$watch通知，代替$watch通知是延迟的知道$digest 阶段。这个延迟是需要的，因为聚合了多次的模型更新到一个 $watch通知里面。而且保证在$watch通知的时候没有其他$watch在运行。如果一个$watch改变了model的值，一次$digest 循环就会强制开始。

1.创建

根scope在应用程序被$injector引导的时候创建。在模板连接的阶段，一些指令可以创建子scope。

2.watcher 注册

在模板连接的时候，指令在scope上面注册了watch 。这些watches用来传播模型值到dom。

3.模型修改

为了变化被检测到，你应该让他们在 apply里面。在控制器里面的同步工作，或者$http,$timeout,$interval里面的工作，系统都隐含的调用了$apply方法。 

4.变化观察器

在$apply的最终阶段，angular会从根scope执行一个$digest 循环一直到遍及孩子scope。
在一个$digest循环中，所有被$watch的表达式都会检查一下模型的修改。如果模型修改了，$watch监听器就会被调用。

5.Scope销毁

当子scope不在需要的时候，子scope的创建者调用scope.$destory()方法来销毁。
这样子scope就不会再进入digest循环，另外子scope的内存也会被垃圾回收器回收。

scopes和directives
指令分为两类
.观察指令(:) 例如{ { } }.用$watch注册了一个回调方法。当数据有变化的时候，这个指令会被通知，来更新视图。
.监听器指令(:) ng-click，当dom 监听器fire的时候，指令执行有关的表达式，并且调用$apply来刷新视图。
当而外的事件（例如用户行为，timer，xhr）,需要调用$apply 方法来触发所有的监听器。创建scope的指令
ng-repeat,ng-controller.创建了新的子scope并且附加新子scope到响应的dom元素上面。

控制器和scopes
控制器和scope以下面的方式交互
.控制器用scope暴露方法到模板
.控制器定义方法修改模型
.控制器注册watcher

scope $watch性能考虑
angular要做大量的脏检查。所有脏检查函数必须又要效率。里面不要访问dom，因为dom访问效率和直接访问js属性的效率差别很大。（成量级的差别。）

$scope watch 深度
.$watch(:)观察了引用
.$watchCollection(:)观察了集合
.$watch(,,true)(:)观察的最细致。每个值都能观察到
用哪一种观察模式，根据情况判断。

和浏览器事件循环集成

事件循环
1.浏览器的事件循环等待一个事件到来。一个事件就是用户的行为、timer事件，网络事件。
2.事件的回调函数会执行。这就进入了js的上下文。这个回调函数可以修改DOM结构
3.一旦回调执行完毕，浏览器离开js的上下文并且开始根据dom的变化重选渲染视图。
