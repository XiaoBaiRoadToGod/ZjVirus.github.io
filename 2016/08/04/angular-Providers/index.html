angular Providers

发表于 2016-08-04   |  
Providers
每个应用程序都是有许多对象相互协作来完成任务的。这些对象需要实例化和包装到一块让app运行。angular中大部分对象被 injector 服务注入和实例化。

注入器创建两种类型的对象：服务和特殊对象。

服务是一个对象上面有许多开发者定义的服务api

特殊对象遵守特定的angular api。这些对象一般属于 控制器，指令，过滤器，动画。

注入器需要知道怎么创建这些对象。你通过注册一个 recipe 来告诉注入器怎么来创建你的对象。有五种recpe类型。

最麻烦的但也是最综合的类型是 provider recipe.剩下的四个是：–value,Factory,Service, Constant–他们四个是provider的语法糖。

Value Recipe
我们需要一个很简单的clientId服务，这个服务只提供一个字符串。

var myApp = angular.module('myApp',[]); myApp.value('clientId','1111');

注意，我们是怎么创建模块，并且指定模块定义包含一个 recipe（用来创建clientid服务）

Factory Recipe
Value reciple写着很简单，但是缺乏很多我们平常需要的重要特征。
Factory recipe添加了下面功能：

可以依赖其他服务 服务初始化
* 懒初始化
一个factory recipe 用一个有0个或多个参数的函数来创建一个服务。这个函数的返回值是这个服务的实例。

注意：在angular中所有的服务都是单例。这意味着注入器使用每种recipe最多一次，之后都放到缓存里面以后用。

factory比value强大。

Service recipe
service recipe和value，factory很像。但是，他调用了一个new 构造函数。

服务的设计模式遵循一个叫 构造器注入的设计模式。

function UnicornLauncher(apiToken) { this.launchedCount = 0; this.launch = function() { // Make a request to the remote API and include the apiToken ... this.launchedCount++; } } myApp.factory('unicornLauncher', ["apiToken", function(apiToken) { return new UnicornLauncher(apiToken); }]);

可以用service这样做
myApp.service('unicornLauncher', ["apiToken", UnicornLauncher]);


Provider Recipe
Provider recipe是核心 recipe。其他所有的recipe都是他的语法糖。他是最麻烦的也是最最强大的，对大部分服务，他就有点大材小用。
Provider recipe 是一个实现了$get方法的自定义类型的语法定义。这个get方法和我们在factory里面定义的一样。实际上，如果你定义了一个factory recipe，一个空的Provider就创建了，上面的$get方法自动设置到了 factory 方法。
只有当你想在应用程序启动之前，给一个应用程序级别的配置暴露一个api。这主要在不同应用程序之间重用的服务配置不同的时候，才有用。
provicer是被注入到config函数的。这个注入是provider 注入器完成。和其他的实例注入器不一样。这个注入器只实例化和包装provider 实例。
在应用程序启动的时候，在angular创建服务之前，他配置和实例化所有的provider。这叫做应用生命周期的配置阶段。在这个阶段中，服务不能访问，他们还没有创建。
一旦配置阶段完成，和provider的交互就不允许了。就开始创建服务。这个阶段叫运行阶段。

Constant Recipe
我们知道angular可以分为配置和运行阶段。在配置阶段，其他服务都没有，连value recipe 的都没有。
由于类似简单的URL地址，不需要什么配置和依赖。他经常可以在配置阶段和运行阶段都可以访问。这个时候Constant 菜谱就有用了。


特殊目的的对象
这些特殊对象和服务不一样。这些对象以插件的形式扩展了框架因此需要实现angular提供的接口。这些接口是控制器，指令，过滤器，animation。

注入器创建这些特殊对象，在幕后，还是用的工厂方法。

总结
  1. 注入器只创建两种类型的对象，服务和特殊类型的对象
  2. 有五种菜谱，value,factory,service,provider,constant
  3. factory和service最常用。service常用在创建一个自定义类型的对象，factory 产生一个函数活基本类型
  4. Provide recipe是最核心的，其他都是他的语法糖。
  5. 所有的特殊对象都是factoryrecipe创建的（除了controller）
